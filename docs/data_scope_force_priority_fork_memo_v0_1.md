## Future triage schema

A future triage should include these columns:

- force / lane;
- Atlas group / cluster;
- data available now;
- observable before move;
- expected forecastable content;
- crowdedness / already-priced risk;
- signal-to-noise risk;
- testable sample density / power;
- governance cost;
- current status;
- next possible action;
- reason to continue / park / close.

Important distinction:

`signal-to-noise risk` asks whether a signal exists and is swamped by noise.

`testable sample density / power` asks whether enough independent observations exist to estimate anything at all.

Both columns are required.

They are not substitutes.
